# cmd/compiler/tests

该目录保存编译器入口的 smoke fixture。这里测试的是“编译阶段”行为，不是运行期沙箱行为。

## 目录约定

- 每个子目录都是一个独立工作目录，通常包含 `compile.json` 和待编译源码。
- 编译输出默认会落在用例目录内的 `compile.out` / `compile.err`。
- 常见运行方式是在具体用例目录下执行 `../../compiler`。

## 现有用例

- `c/`、`cpp/`、`fpc/`：常规语言编译成功路径。
- `16g/`：大内存压力场景。
- `bigexe/`：大体积产物相关场景。
- `build_bomb/`：异常或极端编译负载场景。

## 适合来这里排查的问题

- `compile.json` 默认值是否足够
- 编译阶段的 CPU / 内存 / 输出限制是否生效
- 不同编译器命令行参数是否需要调整

## 不要在这里找的内容

- 运行期 syscall 白名单或沙箱行为不在这里，相关内容请看仓库根目录下的 [`tests/`](../../../tests/) 和 [`runner/`](../../../runner/)。
